JavaScript(자바스크립트) 기초 학습을 위한 단계별 커리큘럼을 체계적으로 정리해 드립니다. 이 커리큘럼은 프로그래밍의 기본 개념에서 시작하여 웹 페이지를 동적으로 제어하는(DOM/이벤트) 과정까지 순차적으로 구성되어 있습니다.


## 1단계: JavaScript 입문 및 기본 환경 설정

JavaScript가 무엇인지 이해하고, 코드를 작성하고 실행할 수 있는 기본 환경을 배웁니다.

* **JavaScript란?**
    * 웹 페이지를 동적으로 만들기 위한 스크립트 언어.
    * 브라우저에서 동작하는 방식 (HTML, CSS와의 관계).
* **개발 환경**
    * VSCode와 같은 코드 에디터 설치.
    * 브라우저 개발자 도구 (특히 **Console** 탭 활용법).
* **JavaScript 코드 삽입 방법**
    * Inline 방식 (HTML 태그 내 `onclick` 등) - *권장하지 않음.*
    * Internal 방식 (`<script>` 태그 사용).
    * External 방식 (`<script src="app.js"></script>` 파일 분리) - **권장.**
* **첫 번째 코드 실행**
    * `console.log()`: 개발자 콘솔에 메시지 출력 (학습 및 디버깅의 핵심).
    * `alert()`: 경고창 띄우기.
* **주석(Comments)**
    * `//` (한 줄 주석), `/* */` (여러 줄 주석).

---

## 2단계: 핵심 문법 - 변수, 자료형, 연산자

데이터를 저장하고 처리하는 프로그래밍의 가장 기본적인 요소를 배웁니다.

* **변수(Variables)**
    * 데이터를 저장하는 공간.
    * `let`: 재할당이 가능한 변수 (ES6+ 표준).
    * `const`: 재할당이 불가능한 상수.
    * (참고) `var`: 오래된 방식 (함수 스코프, 호이스팅 문제) - *사용 지양.*
* **데이터 타입(Data Types)**
    * **원시 타입 (Primitive Types)**
        * `String` (문자열): `""`, `''`, `` ` `` (템플릿 리터럴).
        * `Number` (숫자): 정수, 실수, `NaN`, `Infinity`.
        * `Boolean` (논리): `true`, `false`.
        * `undefined`: 값이 할당되지 않음.
        * `null`: 의도적으로 '없음'을 명시.
    * **객체 타입 (Object Type)**
        * `Object` (객체): `{}`, `Array` (배열): `[]`, `Function` (함수).
* **연산자(Operators)**
    * **산술**: `+`, `-`, `*`, `/`, `%` (나머지), `**` (거듭제곱).
    * **할당**: `=`, `+=`, `-=`, `*=`, `/=`, `%=`.
    * **비교**: `==` (값 비교) vs. `===` (값과 타입 비교) - **`===` 사용 권장.**
    * `!=`, `!==`, `>`, `<`, `>=`, `<=`.
    * **논리**: `&&` (AND), `||` (OR), `!` (NOT).
    * **증감**: `++` (1 증가), `--` (1 감소).
    * **삼항 연산자**: `(조건) ? (true일 때) : (false일 때)`.

---

## 3단계: 제어 흐름 - 조건문과 반복문

코드의 실행 순서를 제어하고 논리적인 흐름을 만듭니다.

* **조건문(Conditional Statements)**
    * `if`, `else if`, `else`: 특정 조건에 따라 다른 코드를 실행.
    * `switch...case`: 여러 값 중 하나를 선택하여 실행.
* **반복문(Loops)**
    * `for`: 정해진 횟수만큼 코드를 반복 (가장 일반적).
    * `while`: 특정 조건이 `true`인 동안 코드를 반복.
    * `do...while`: 코드를 먼저 한 번 실행하고, 조건이 `true`인 동안 반복.
* **반복문 제어**
    * `break`: 현재 반복문을 즉시 종료.
    * `continue`: 현재 반복을 건너뛰고 다음 반복으로 진행.

---

## 4단계: 데이터 구조 - 객체와 배열

관련된 데이터를 묶어서 효율적으로 관리하는 방법을 배웁니다.

* **객체(Objects)**
    * Key(키)와 Value(값) 쌍으로 이루어진 데이터의 집합.
    * 객체 리터럴: `{ name: "홍길동", age: 30 }`.
    * 속성(Property) 접근: `object.name` (Dot notation), `object['name']` (Bracket notation).
    * 메서드(Method): 객체에 속한 함수.
* **배열(Arrays)**
    * 순서가 있는 데이터의 목록.
    * 배열 리터럴: `[10, "A", true]`.
    * 인덱스(Index): `0`부터 시작하는 요소의 순서 (e.g., `array[0]`).
    * 주요 속성 및 메서드:
        * `.length`: 배열의 길이.
        * `push()` / `pop()`: 배열 끝에 요소 추가/제거.
        * `shift()` / `unshift()`: 배열 앞에 요소 제거/추가.
        * `slice()` / `splice()`: 배열 자르기 및 수정.

---

## 5단계: 재사용 - 함수(Functions)

특정 작업을 수행하는 코드 블록을 정의하고 필요할 때마다 호출하여 사용합니다.

* **함수의 필요성**: 코드의 재사용, 모듈화.
* **함수 정의**
    * **함수 선언문**: `function sayHello() { ... }`.
    * **함수 표현식**: `const sayHello = function() { ... };`.
* **매개변수(Parameters)와 인자(Arguments)**
    * 함수 정의 시 받는 값 (매개변수) vs. 함수 호출 시 전달하는 값 (인자).
* **반환(Return)**
    * `return` 키워드: 함수 실행 결과를 외부로 반환.
* **스코프(Scope)**
    * **전역 스코프(Global Scope)**: 코드 어디에서나 접근 가능.
    * **지역 스코프(Local Scope)**: 함수 내부에서만 접근 가능 (ES6 이후 **블록 스코프** `let/const` 포함).
* **(ES6+) 화살표 함수(Arrow Functions)**
    * `const add = (a, b) => a + b;`
    * 간결한 문법 (단, `this` 바인딩 방식이 다름).

---

## 6단계: 웹 페이지 제어 (DOM)

JavaScript를 사용하여 HTML 문서의 요소를 선택하고, 내용을 변경하거나 스타일을 조작합니다.

* **DOM(Document Object Model)이란?**
    * 브라우저가 HTML 문서를 해석하여 만든 객체 트리 구조.
* **HTML 요소 선택(Selecting)**
    * `document.getElementById('id-name')`
    * `document.getElementsByClassName('class-name')`
    * `document.getElementsByTagName('tag-name')`
    * **`document.querySelector('CSS 선택자')`** (단일 요소 선택) - **권장.**
    * **`document.querySelectorAll('CSS 선택자')`** (여러 요소 선택) - **권장.**
* **HTML 요소 조작(Manipulating)**
    * 내용 변경: `element.textContent`, `element.innerHTML`.
    * 스타일 변경: `element.style.color = 'red';`, `element.style.fontSize = '16px';`.
    * 속성 변경: `element.setAttribute()`, `element.getAttribute()`.
    * 클래스 조작: `element.classList.add()`, `element.classList.remove()`, `element.classList.toggle()`.
* **새로운 요소 생성 및 추가**
    * `document.createElement('div')`
    * `parentElement.appendChild(childElement)`
    * `element.remove()`

---

## 7단계: 사용자 반응 (Event)

사용자의 행동(클릭, 키보드 입력 등)에 반응하여 JavaScript 코드를 실행합니다.

* **이벤트(Event)란?**: 사용자의 클릭, 마우스 이동, 키보드 입력, 페이지 로드 등.
* **이벤트 핸들러(Event Handler) 등록**
    * `element.addEventListener('이벤트 타입', '실행할 함수')` - **권장.**
    * (참고) HTML 속성 (`onclick="..."`)이나 JS 속성 (`element.onclick = ...`)은 구식.
* **주요 이벤트 타입**
    * **마우스**: `click`, `mouseover`, `mouseout`, `mousedown`, `mouseup`.
    * **키보드**: `keydown`, `keyup`, `keypress`.
    * **폼**: `submit` (폼 제출), `change` (입력 값 변경).
    * **문서**: `load` (페이지 로드 완료), `DOMContentLoaded` (DOM 파싱 완료).
* **이벤트 객체(Event Object)**
    * 이벤트 발생 시 핸들러 함수로 전달되는 객체.
    * `event.target`: 이벤트가 실제로 발생한 요소.
    * `event.preventDefault()`: 이벤트의 기본 동작(e.g., 폼 제출)을 막음.


## 1단계: jQuery 소개 및 설정

jQuery의 기본 개념과 사용 환경을 설정합니다.

* **jQuery란?**
    * JavaScript 라이브러리임을 이해 (jQuery는 새로운 언어가 아님).
    * DOM 조작, 이벤트 처리, 애니메이션, Ajax를 단순화.
* **jQuery 설치(포함) 방법**
    * **CDN (Content Delivery Network)** 사용: `<script>` 태그 한 줄로 HTML에 포함 (권장).
    * (예: ` <script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>`)
    * 파일 다운로드: jQuery 파일을 직접 다운로드하여 프로젝트에 포함.
* **`$(document).ready()`**
    * jQuery 코드 실행의 시작점.
    * HTML 문서(DOM)가 완전히 로드된 후 스크립트를 실행하도록 보장.
    * 단축형: `$(function() { ... });`

---

## 2단계: 핵심 - 선택자(Selectors)

jQuery의 가장 강력한 기능. CSS 선택자를 활용하여 HTML 요소를 매우 쉽게 선택합니다.

* **기본 선택자**
    * `$("태그이름")`: 예) `$("p")`
    * `$("#아이디")`: 예) `$("#myId")`
    * `$(".클래스")`: 예) `$(".myClass")`
* **복합 선택자**
    * `$("A B")`: A의 자손 B
    * `$("A > B")`: A의 직계 자식 B
    * `$("A + B")`: A의 바로 다음 형제 B
    * `$("A, B")`: A와 B 모두
* **속성 선택자**
    * `$("A[속성='값']")`: 예) `$("input[type='text']")`
* **필터 선택자 (jQuery 고유)**
    * `:first`, `:last`, `:even` (짝수), `:odd` (홀수)
    * `:eq(index)` (n번째 요소), `:gt(index)` (n보다 큰 요소), `:lt(index)` (n보다 작은 요소)
    * `:has(selector)` (특정 요소를 포함하는 요소)
* **폼 선택자**
    * `:input`, `:text`, `:password`, `:radio`, `:checked`

---

## 3단계: DOM 조작 (1) - 내용 및 속성

선택한 요소의 내용, 값, 속성을 변경합니다.

* **내용(Content) 변경**
    * `.text()`: 요소의 텍스트 내용만 가져오거나 설정 (JavaScript의 `textContent`).
    * `.html()`: 요소의 HTML 내용까지 가져오거나 설정 (JavaScript의 `innerHTML`).
* **폼(Form) 값 변경**
    * `.val()`: `input`, `select`, `textarea`의 `value` 값을 가져오거나 설정.
* **속성(Attribute) 변경**
    * `.attr('속성명')`: 속성값 가져오기.
    * `.attr('속성명', '새 값')`: 속성값 설정.
    * `.removeAttr('속성명')`: 속성 제거.

---

## 4단계: DOM 조작 (2) - 요소 추가 및 제거

HTML 문 T서에 새로운 요소를 동적으로 생성, 추가, 이동, 제거합니다.

* **새 요소 생성**: `$("<div>새로운 내용</div>")`
* **요소 추가 (내부)**
    * `.append(내용)`: 선택한 요소의 **마지막 자식**으로 추가.
    * `.prepend(내용)`: 선택한 요소의 **첫 번째 자식**으로 추가.
* **요소 추가 (외부)**
    * `.after(내용)`: 선택한 요소의 **바로 뒤**에 추가.
    * `.before(내용)`: 선택한 요소의 **바로 앞**에 추가.
* **요소 제거**
    * `.remove()`: 선택한 요소 **자신을** DOM에서 제거.
    * `.empty()`: 선택한 요소의 **모든 자식 요소를** 제거 (자신은 남음).

---

## 5단계: CSS 조작 및 스타일

JavaScript로 스타일을 제어하는 복잡함을 jQuery로 단순화합니다.

* **CSS 스타일 직접 제어**
    * `.css('속성명')`: CSS 속성값 가져오기.
    * `.css('속성명', '값')`: CSS 속성값 설정.
    * `.css({ '속성1': '값1', '속성2': '값2' })`: 여러 속성 동시 설정 (매우 유용).
* **CSS 클래스 제어** (JavaScript의 `classList`와 유사)
    * `.addClass('클래스명')`
    * `.removeClass('클래스명')`
    * `.toggleClass('클래스명')` (있으면 제거, 없으면 추가)
    * `.hasClass('클래스명')`: 클래스 유무 확인 (true/false 반환).

---

## 6단계: 이벤트 핸들링

사용자 상호작용(이벤트)을 처리합니다.

* **이벤트 등록의 기본: `.on()`** (JavaScript의 `addEventListener` 역할)
    * `$(선택자).on('이벤트타입', function() { ... });`
    * 예: `$("button").on('click', function() { alert('클릭됨!'); });`
* **자주 쓰는 이벤트 단축 메서드**
    * `.click()`, `.dblclick()`
    * `.mouseover()`, `.mouseout()`, `.hover()` (mouseover와 mouseout을 합친 것)
    * `.keydown()`, `.keyup()`, `.keypress()`
    * `.change()` (`select`, `input` 값 변경 시)
    * `.submit()` (폼 제출 시)
* **`$(this)`의 이해**
    * 이벤트 핸들러 함수 내에서 `$(this)`는 **이벤트가 발생한 바로 그 요소**를 가리킵니다.
* **이벤트 기본 동작 중지**
    * `event.preventDefault()`: (함수의 매개변수로 `event`를 받아 사용) 폼 제출이나 링크 이동을 막을 때 사용.

---

## 7단계: 효과(Effects) 및 애니메이션

jQuery가 초기에 큰 인기를 얻은 이유. 간단한 코드로 부드러운 시각 효과를 구현합니다.

* **기본 효과**
    * `.show()`, `.hide()`, `.toggle()`: 요소를 보이거나 숨김.
* **페이드(Fading) 효과**
    * `.fadeIn(speed)`, `.fadeOut(speed)`, `.fadeToggle(speed)`
* **슬라이드(Sliding) 효과**
    * `.slideDown(speed)`, `.slideUp(speed)`, `.slideToggle(speed)`
* **커스텀 애니메이션**
    * `.animate({ CSS 속성 }, speed, callback)`
    * 예: `$("#box").animate({ left: '250px', opacity: '0.5' }, 500);`

---

## 8단계: Ajax - 비동기 통신

서버와 데이터를 비동기적으로 주고받습니다. (JavaScript의 `fetch`나 `XMLHttpRequest`를 극도로 단순화)

* **`$.ajax()`**: 가장 기본적이고 모든 설정을 할 수 있는 메서드.
* **단축 메서드 (더 많이 사용)**
    * `$.get(URL, function(data) { ... });`: GET 방식으로 서버에서 데이터 가져오기.
    * `$.post(URL, { 전송할 데이터 }, function(response) { ... });`: POST 방식으로 서버에 데이터 전송하기.
    * `$.getJSON()`: JSON 데이터만 전문적으로 가져오기.



Bootstrap(부트스트랩)은 **반응형, 모바일 우선** 웹사이트를 빠르고 쉽게 만들 수 있도록 도와주는 **CSS 프레임워크**입니다. 미리 디자인된 CSS 스타일과 JavaScript 기반의 상호작용 컴포넌트들을 제공합니다.

JavaScript와 jQuery를 학습하셨으니, Bootstrap을 통해 얼마나 빠르고 효율적으로 웹 페이지의 레이아웃과 디자인을 구성할 수 있는지 배우실 차례입니다.

**중요: Bootstrap 5 (최신 버전) vs. Bootstrap 4 (이전 버전)**

* **Bootstrap 5 (현재 표준):** **jQuery가 필요 없습니다.** 순수 JavaScript(Vanilla JS)를 사용합니다. 지금 새로 배운다면 5버전으로 시작하는 것을 강력히 권장합니다.
* **Bootstrap 4 (레거시):** **jQuery가 필수입니다.** 모든 JavaScript 컴포넌트(모달, 드롭다운 등)가 jQuery에 의존합니다.


## 1단계: Bootstrap 시작하기 (환경 설정)

Bootstrap을 웹 페이지에 적용하고 기본 개념을 이해합니다.

* **Bootstrap이란?**
    * CSS/JavaScript 프레임워크의 개념.
    * "반응형 웹"과 "모바일 우선"의 의미.
* **Bootstrap 추가하기 (CDN 방식)**
    * HTML 파일 `<head>` 태그 내에 CSS 링크 추가.
    * HTML `<body>` 태그 닫기 직전에 JavaScript 번들(`<script>`) 태그 추가.
* **필수 메타 태그**
    * 반응형 동작을 위한 `viewport` 메타 태그 설정.
    * ` <meta name="viewport" content="width=device-width, initial-scale=1"> `
* **기본 HTML 템플릿 (Starter Template)**
    * 공식 문서의 기본 템플릿 구조 이해.

---

## 2단계: 핵심 레이아웃 - 그리드 시스템 (Grid System)

Bootstrap의 가장 중요하고 강력한 기능. 웹 페이지의 레이아웃을 12개의 열(Column)로 나누어 반응형으로 배치합니다.

* **` .container ` vs `.container-fluid`**
    * 고정된 너비 (좌우 여백 O) vs. 전체 너비 (좌우 여백 X).
* **` .row ` (행)**
    * 컬럼(열)을 감싸는 부모 역할.
* **` .col-* ` (열)**
    * `.col`: 12개를 자동으로 나눠 가짐 (예: 3개면 4/4/4).
    * `.col-6`: 12개 중 6개(절반) 차지.
    * `.col-auto`: 내용물만큼 너비 차지.
* **반응형 중단점 (Breakpoints)**
    * `sm` (모바일), `md` (태블릿), `lg` (데스크탑), `xl` (대형 데스크탑).
    * 예: ` <div class="col-12 col-md-6 col-lg-4"> `
        * 모바일(sm)에서는 12칸(100%).
        * 태블릿(md)부터는 6칸(50%).
        * 데스크탑(lg)부터는 4칸(33.3%).

---

## 3단계: 콘텐츠(Content) 스타일링

기본적인 HTML 태그(글자, 이미지, 테이블 등)를 Bootstrap 스타일로 꾸밉니다.

* **타이포그래피 (Typography)**
    * 기본 글꼴 및 크기.
    * Heading (`h1`~`h6`), Display 클래스 (`.display-1`).
    * `.lead` (강조 문단), `<blockquote>`.
* **이미지 (Images)**
    * `.img-fluid`: 부모 요소 너비에 맞게 100% 반응형 이미지 (필수).
    * `.img-thumbnail`: 썸네일 테두리.
* **테이블 (Tables)**
    * `.table`: 기본 테이블 스타일.
    * `.table-striped` (줄무늬), `.table-hover` (마우스 오버).

---

## 4단계: 폼(Forms) 컨트롤

사용자 입력을 받는 `form` 요소들을 깔끔하게 디자인합니다.

* **기본 폼 컨트롤**
    * `.form-control`: `<input>`, `<textarea>`, `<select>`에 적용 (100% 너비).
* **폼 레이아웃**
    * `.form-label`: 라벨 스타일.
    * `.form-text`: 도움말 텍스트.
* **체크박스와 라디오**
    * `.form-check`, `.form-check-input`, `.form-check-label`.
* **인라인 폼 및 폼 그리드**
    * 그리드 시스템(`row`, `col`)을 활용한 폼 배치.

---

## 5단계: 컴포넌트 (Components) - (1) 기본 UI

Bootstrap이 미리 만들어 둔 재사용 가능한 UI 조각들입니다. (JavaScript 불필요)

* **버튼 (Buttons)**
    * `.btn`, `.btn-primary` (주색상), `.btn-secondary` (보조색상).
    * `.btn-lg` (큰 버튼), `.btn-sm` (작은 버튼).
* **카드 (Cards)**
    * 가장 많이 쓰이는 범용 컨테이너.
    * `.card`, `.card-body`, `.card-title`, `.card-text`.
    * `.card-header`, `.card-footer`, `.card-img-top`.
* **알림 (Alerts)**
    * `.alert`, `.alert-success` (성공), `.alert-danger` (위험).
* **뱃지 (Badges)**
    * `.badge`, `.bg-primary` (5버전부터 `.bg-*` 사용).
* **리스트 그룹 (List groups)**
    * `.list-group`, `.list-group-item`.

---

## 6단계: 컴포넌트 (Components) - (2) JavaScript 기반

JavaScript(BS5) 또는 jQuery(BS4)가 있어야 동작하는 상호작용형 컴포넌트입니다.

* **네비게이션 바 (Navbar)**
    * 웹사이트의 상단 메뉴 (매우 중요).
    * `.navbar`, `.navbar-brand`, `.navbar-nav`, `.nav-item`, `.nav-link`.
    * 반응형 토글 버튼 (`.navbar-toggler`).
* **드롭다운 (Dropdowns)**
    * 버튼 클릭 시 메뉴가 펼쳐짐.
* **모달 (Modals)**
    * 화면 위에 뜨는 팝업 레이어.
* **캐러셀 (Carousel)**
    * 이미지 슬라이드.
* **어코디언 (Accordion / Collapse)**
    * 클릭 시 내용이 접혔다 펴지는 메뉴.
* **작동 원리 (중요)**
    * HTML 태그에 `data-bs-toggle="modal"` , `data-bs-target="#myModal"` 같은 **데이터 속성(Data Attributes)**을 작성하는 것만으로 대부분 동작.

---

## 7단계: 유틸리티 (Utilities)

그리드나 컴포넌트만으로 해결하기 힘든 세밀한 디자인 조정을 위한 '헬퍼 클래스'입니다. (매우 강력)

* **간격 (Spacing)** - **(필수)**
    * `m-*` (margin), `p-*` (padding).
    * `t` (top), `b` (bottom), `s` (start - left), `e` (end - right), `x` (x축), `y` (y축).
    * 숫자: `0` (0) ~ `5` (3rem), `auto`.
    * 예: `mt-3` (margin-top: 1rem), `px-5` (padding-left/right: 3rem).
* **색상 (Colors)**
    * `.text-primary`, `.text-danger` (글자색).
    * `.bg-dark`, `.bg-light` (배경색).
* **Flexbox 유틸리티**
    * `.d-flex` (display: flex).
    * `.justify-content-*`, `.align-items-*`.
* **보이기/숨기기 (Display)**
    * `.d-none` (숨김).
    * `.d-md-block` (md 사이즈 이상에서만 보임).

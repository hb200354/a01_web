## 🎮 JAVA 2단계: 미션 컨트롤 센터 (총 15문제)

**테마: 🛡️ 전설의 용사, 코드의 미궁을 탈출하라\! 🛡️**
당신은 '코드의 미궁'에 갇힌 용사입니다. 각 방을 탈출하려면 `boolean` (true/false)으로 답해야 하는 마법의 문을 통과해야 합니다.

### 1\. 🧊 얼음 동굴 구역 (Relational Operators)

**문제 1: 흔들리는 다리**

  * 다리를 건너려면 당신의 무게(`myWeight` = 70)와 당신이 든 보석(`jewelWeight` = 10)의 합이, 다리의 최대 하중(`bridgeLimit` = 100) '미만'이어야 합니다.
  * **미션:** 다리를 건널 수 있는지(`true`/`false`) `boolean` 변수 `canCross`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] `myWeight`와 `jewelWeight`를 더하는 `+` 연산자를 사용함.
      * [B] '미만'을 의미하는 `<` 비교 연산자를 정확히 사용함.
      * [C] `(70 + 10) < 100`의 결과인 `true`가 정확히 출력됨.

**문제 2: 쌍둥이 골렘**

  * 두 골렘(`golem1` = 50, `golem2` = 50)이 지키는 문이 있습니다. 두 골렘의 힘이 '완전히 같을 때만' 문이 열립니다.
  * **미션:** 문이 열리는지(`true`/`false`) `boolean` 변수 `isOpen`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] '같다'를 의미하는 `==` 비교 연산자를 사용함 (절대 `=` 사용 안 함).
      * [B] `50 == 50`의 결과인 `true`가 정확히 출력됨.

**문제 3: 가짜 보물 상자**

  * 진짜 보물 상자(`realChest` = 1)와 가짜 보물 상자(`fakeChest` = 2)가 있습니다. 두 상자의 ID가 '서로 다를 때'만 함정이 발동합니다.
  * **미션:** 함정이 발동하는지(`true`/`false`) `boolean` 변수 `isTrapActivated`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] '다르다'를 의미하는 `!=` 비교 연산자를 사용함.
      * [B] `1 != 2`의 결과인 `true`가 정확히 출력됨.

**문제 4: 드래곤의 불꽃**

  * 드래곤을 이기려면, 내 '얼음 마법' 레벨(`iceMagic` = 100)이 드래곤의 '화염 저항'(`fireResist` = 99)보다 '크거나 같아야' 합니다.
  * **미션:** 드래곤을 이길 수 있는지(`true`/`false`) `boolean` 변수 `canWin`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] '크거나 같다'(이상)를 의미하는 `>=` 비교 연산자를 사용함.
      * [B] `100 >= 99`의 결과인 `true`가 정확히 출력됨.

**문제 5: 독 안개 복도**

  * 독 안개 복도를 통과하려면, 내 '해독 물약' 개수(`antidote` = 5)가 필요한 개수인 `requiredPotion` = 5와 '같거나 적어야' 안전합니다. (5개까지만 허용)
  * **미션:** 안전하게 통과할 수 있는지(`true`/`false`) `boolean` 변수 `isSafe`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] '작거나 같다'(이하)를 의미하는 `<=` 비교 연산자를 사용함.
      * [B] `5 <= 5`의 결과인 `true`가 정확히 출력됨.

### 2\. 🔥 불의 미궁 구역 (Logical Operators)

**문제 6: 두 개의 압력 발판 (&&)**

  * 문이 열리려면 1번 발판(`plate1` = `true`) **그리고** 2번 발판(`plate2` = `true`)을 '동시에' 밟아야 합니다.
  * **미션:** `plate1`은 밟았지만(`true`), `plate2`는 밟지 못했을 때(`false`), 문이 열리는지 `boolean` 변수 `isDoorOpen`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] '그리고'(AND)를 의미하는 `&&` 논리 연산자를 사용함.
      * [B] `true && false`의 결과인 `false`가 정확히 출력됨.

**문제 7: 마법사 또는 전사 (||)**

  * 파티에 '마법사'(`isWizard` = `false`)가 있거나 **또는** '전사'(`isWarrior` = `true`)가 있으면, 보너스 버프를 받습니다.
  * **미션:** 우리 파티가 보너스 버프를 받는지(`true`/`false`) `boolean` 변수 `getCoin`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] '또는'(OR)을 의미하는 `||` 논리 연산자를 사용함.
      * [B] `false || true`의 결과인 `true`가 정확히 출력됨.

**문제 8: 투명화 물약 (\!)**

  * '투명화 물약'(`isInvisible` = `true`)을 마시면 경비병에게 보이지 않아야 합니다. 경비병의 시야(`canSee`)는 `!isInvisible` 입니다.
  * **미션:** 경비병이 나를 볼 수 있는지(`true`/`false`) `boolean` 변수 `canSee`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] '반대'(NOT)를 의미하는 `!` 논리 연산자를 사용함.
      * [B] `!true`의 결과인 `false`가 정확히 출력됨.

**문제 9: 청개구리 경비병 (\!)**

  * 경비병(`isGuard`)은 `false` (없음) 상태입니다. 하지만 청개구리 마법에 걸려 "경비병이 '없는 것'이 '아니다'"라고 `boolean`에 저장해야 합니다.
  * **미션:** `boolean isGuard = false;`를 `!isGuard`로 반전시켜 `boolean finalCheck`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] `!` 연산자를 `false` 값에 적용함.
      * [B] `!false`의 결과인 `true`가 정확히 출력됨.

### 3\. 💎 최종 보스 구역 (Complex Operators)

**문제 10: 보스 방의 첫 번째 문 (&&와 \>)**

  * 보스 방에 들어가려면 '내 레벨'(`myLevel` = 30)이 25 '초과'여야 하고 **(AND)** '황금 열쇠'(`hasGoldKey` = `true`)를 가지고 있어야 합니다.
  * **미션:** 첫 번째 문을 통과할 수 있는지(`true`/`false`) `boolean` 변수 `passFirstGate`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] `>` 비교 연산자와 `&&` 논리 연산자를 함께 사용함.
      * [B] `(30 > 25) && (hasGoldKey == true)`의 논리 구조를 완성함.
      * [C] `(true && true)`의 결과인 `true`가 정확히 출력됨.

**문제 11: 보스 방의 두 번째 문 (||와 ==)**

  * 두 번째 문은 '파티원 수'(`partySize` = 4)가 4명과 '같거나' **(OR)** '길드 마스터'(`isGuildMaster` = `false`)여야 통과됩니다.
  * **미션:** 두 번째 문을 통과할 수 있는지(`true`/`false`) `boolean` 변수 `passSecondGate`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] `==` 비교 연산자와 `||` 논리 연산자를 함께 사용함.
      * [B] `(partySize == 4) || (isGuildMaster == true)`의 논리 구조를 완성함.
      * [C] `(true || false)`의 결과인 `true`가 정확히 출력됨.

**문제 12: 함정 발동 조건 (&&와 ||의 혼합)**

  * 함정은 ( '무게'(`weight` = 80)가 100 '초과'이거나 **(OR)** '타이머'(`timeLimit` = 50)가 60 '미만' ) **(AND)** '경보'(`isAlert` = `true`)가 울릴 때 발동합니다.
  * **미션:** 함정이 발동하는지(`true`/`false`) `boolean` 변수 `isTrap`에 저장하고 출력하세요. (연산자 우선순위를 위해 괄호 `()`를 잘 사용하세요\!)
  * **평가 기준 (Rubric):**
      * [A] `()`, `>`, `<`, `||`, `&&` 연산자를 모두 복합적으로 사용함.
      * [B] `( (weight > 100) || (timeLimit < 60) ) && (isAlert == true)`의 논리 구조를 완성함.
      * [C] `( (false || true) && true )`의 결과인 `true`가 정확히 출력됨.

**문제 13: 보물의 봉인 (\!와 &&)**

  * 보물의 봉인은 ( '달빛'(`isMoonlight` = `false`)이 비치지 않고 **(AND)** '새벽'(`isDawn` = `true`)일 때 ) 풀립니다.
  * **미션:** '달빛이 비치지 않는다'는 `!isMoonlight`로 표현해야 합니다. 봉인이 풀리는지(`true`/`false`) `boolean` 변수 `isSealReleased`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] `!` 연산자와 `&&` 연산자를 복합적으로 사용함.
      * [B] `(!isMoonlight) && (isDawn == true)`의 논리 구조를 완성함.
      * [C] `( (!false) && true )` 즉 `(true && true)`의 결과인 `true`가 정확히 출력됨.

**문제 14: 탈출 포탈 (\!와 ||)**

  * 탈출 포탈은 ( '보스'(`isBossDead` = `true`)가 죽었거나 **(OR)** '비상 스위치'(`isEmergency` = `false`)가 켜졌을 때 ) '아니면' 열리지 않습니다. 즉, 이 두 조건이 `true`가 '아닐 때'(`!`) 열립니다.
  * **미션:** `!(A || B)` 구조를 사용하여 포탈이 열리는지(`true`/`false`) `boolean` 변수 `isPortalOpen`에 저장하고 출력하세요.
  * **평가 기준 (Rubric):**
      * [A] `!` 연산자와 `||` 연산자를 괄호와 함께 복합적으로 사용함.
      * [B] `!(isBossDead || isEmergency)`의 논리 구조를 완성함.
      * [C] `!(true || false)` 즉 `!(true)`의 결과인 `false`가 정확히 출력됨.

**문제 15: 최종 미션 (총 복습)**

  * 미궁을 완전히 탈출하는 조건은 다음과 같습니다.
    1.  ( '내 레벨'(`myLevel` = 30)이 '보스 레벨'(`bossLevel` = 30) '이상' **(AND)** '보스'(`isBossDead` = `true`)가 죽었음 )
    2.  **또는 (OR)**
    3.  ( '숨겨진 문'(`hasSecretDoorKey` = `false`) 열쇠를 가졌거나 **(OR)** '맵'(`hasMap` = `true`)을 가짐 ) **(AND)** '경비병'(`isGuard` = `false`)이 '없음'
  * **미션:** 탈출 여부(`true`/`false`)를 `boolean` 변수 `isEscaped`에 저장하고 출력하세요. (괄호를 매우 조심해서 사용하세요\!)
  * **평가 기준 (Rubric):**
      * [A] `()`, `>=`, `==`, `&&`, `||`, `!`(또는 `== false`) 연산자를 모두 활용함.
      * [B] `( (myLevel >= bossLevel) && isBossDead ) || ( (hasSecretDoorKey || hasMap) && !isGuard )` 구조를 완성함.
      * [C] `( (true && true) || ( (false || true) && true ) )` 즉 `( true || true )`의 결과인 `true`가 정확히 출력됨.

## 🚀 HTML/CSS 2단계: 우주 탐사선 대시보드 (총 15문제)

**테마: 🌌 스페이스 에이전시 "코드-X"의 웹사이트를 구축하라\! 🌌**
당신은 '코드-X' 에이전시의 프론트엔드 개발자입니다. 투박한 HTML 문서를 세련된 우주 탐사선 대시보드로 업그레이드해야 합니다.

### 1\. 🛰️ 위성 통신망 (Table Styling)

**문제 1: 통신 기록표 (Collapse & Padding)**

  * '통신 기록' `<table>`이 '이중 테두리 감옥'처럼 보이고, 글자가 테두리에 붙어 답답합니다.
  * **미션:**
    1.  `table`에 `border-collapse: collapse;`를 적용해 '감옥'을 '합체'시키세요.
    2.  `th`와 `td`에 `padding: 10px;`를 적용해 '숨 쉴 공간'을 주세요.
  * **평가 기준 (Rubric):**
      * [A] `border-collapse: collapse;` 속성을 `table`에 정확히 적용함.
      * [B] `padding: 10px;` 속성을 `th`, `td`에 정확히 적용함.
      * [C] 시각적으로 단일선 테두리와 내부 여백이 구현됨.

**문제 2: 행성 데이터 (Width & Thead)**

  * '행성 데이터' `<table>`가 화면 크기에 상관없이 작게 뭉쳐있습니다.
  * **미션:**
    1.  `table`에 `width: 100%;`를 적용해 대시보드에 꽉 차게 만드세요.
    2.  HTML 구조를 수정하여 제목 행(`<tr>`)을 `<thead>`로, 본문 행들을 `<tbody>`로 감싸세요.
  * **평가 기준 (Rubric):**
      * [A] `width: 100%;` 속성을 `table`에 적용함.
      * [B] HTML 구조에 `<thead>`와 `<tbody>` 태그를 시맨틱하게 적용함.
      * [C] 테이블이 부모 요소 너비에 꽉 차고, 구조가 분리됨.

**문제 3: 대원 명단 (Zebra Striping)**

  * '탐사 대원 명단' `<table>`의 `<tbody>`에 10명의 대원이 나열되어 가독성이 떨어집니다.
  * **미션:** `tbody` 안의 `tr` 중에서 '짝수 번째'(even) 행에만 `background-color: #2c3e50;` (어두운 남색) 배경을 적용하여 '얼룩말 무늬'를 만드세요. (참고: 기본 글자색은 흰색이라고 가정)
  * **평가 기준 (Rubric):**
      * [A] `:nth-child(even)` 가상 클래스 선택자를 정확히 사용함.
      * [B] 선택자를 `tbody tr:nth-child(even)`로 정확히 조합함.
      * [C] 짝수 행에만 배경색이 적용됨.

**문제 4: 긴급 경보 (Hover & Specific Border)**

  * '경보 목록' `<table>`에서 마우스를 올린 행을 강조하고 싶습니다.
  * **미션:**
    1.  `tbody tr`에 마우스를 올리면(`:hover`) `background-color: #e74c3c;` (빨간색)로 변경하세요.
    2.  `thead`의 `th`에는 '아래쪽 테두리'만 `border-bottom: 3px solid #3498db;` (파란색)로 두껍게 만드세요.
  * **평가 기준 (Rubric):**
      * [A] `:hover` 가상 클래스를 `tbody tr`에 적용함.
      * [B] `border-bottom` 속성을 `thead th`에 정확히 적용함.
      * [C] 마우스 오버 시 행 색상이 변하고, 제목 행 하단에만 두꺼운 선이 생김.

**문제 5: 자원 현황판 (Complex Table)**

  * '자원 현황' `<table>`을 총체적으로 디자인합니다.
  * **미션:**
    1.  `border-collapse`, `width: 100%` 적용.
    2.  `thead th`는 '가운데 정렬'(`text-align: center;`)과 '파란 배경'(`background-color: #3498db;`) 적용.
    3.  `tbody td`는 '왼쪽 정렬'(`text-align: left;`)과 `padding: 12px;` 적용.
    4.  `tbody tr`의 '홀수 번째'(odd) 행에 `background-color: #ecf0f1;` (연한 회색) 적용.
  * **평가 기준 (Rubric):**
      * [A] `thead th`와 `tbody td`에 각각 다른 스타일(정렬, 패딩)을 적용함.
      * [B] `:nth-child(odd)`를 사용하여 홀수 행을 정확히 선택함.
      * [C] 문제의 4가지 요구사항이 모두 시각적으로 구현됨.

### 2\. 📬 관제 센터 보고서 (Form Styling)

**문제 6: 폭주하는 보고서 양식 (Box-sizing)**

  * '보고서' `input`에 `width: 100%;`와 `padding: 10px;`를 동시에 줬더니, `padding`만큼 폼이 밖으로 삐져나갔습니다.
  * **미션:** 모든 요소(`*`)에 `box-sizing: border-box;`를 적용하여, `padding`을 줘도 너비(width) 안에서 해결되도록 '초기화'하세요.
  * **평가 기준 (Rubric):**
      * [A] `*` (전체 선택자)를 사용함.
      * [B] `box-sizing: border-box;` 속성을 정확히 적용함.
      * [C] `input`이 `padding`을 가져도 부모 너비(100%)를 넘지 않음.

**문제 7: 관공서 서류 (Width & Margin)**

  * '대원 ID', '비밀 암호' `input`이 너무 작고 서로 붙어있습니다.
  * **미션:** (`box-sizing`이 적용됐다고 가정)
    1.  `input` 요소에 `width: 100%;`를 적용해 '꽉 채우세요'.
    2.  `input` 요소에 `margin-bottom: 15px;`를 줘서 '밖에서 거리'를 두게 하세요.
  * **평가 기준 (Rubric):**
      * [A] `width: 100%;`를 적용하여 `input`이 꽉 차게 만듦.
      * [B] `margin-bottom`을 적용하여 요소 간의 수직 간격을 확보함.
      * [C] 두 `input`이 세로로 정돈되어 보임.

**문제 8: 친절한 보고서 (Label & Padding)**

  * `input`의 `label` 태그가 `input` 옆에 붙어있어 모바일에서 보기 힘듭니다.
  * **미션:**
    1.  `label` 태그에 `display: block;`을 적용해 `input`과 다른 줄을 쓰게 만드세요.
    2.  `input`에 `padding: 12px;`를 줘서 '안쪽 숨쉬기' 공간을 확보하세요.
  * **평가 기준 (Rubric):**
      * [A] `label`에 `display: block;`을 적용함.
      * [B] `input`에 `padding`을 적용함.
      * [C] 레이블이 위, 인풋이 아래로 배치되고 인풋 내부에 여백이 생김.

**문제 9: 집중\! 보고서 작성 (:focus)**

  * '대원 ID' `input`을 클릭해도 사용자가 어디를 입력 중인지 티가 안 납니다.
  * **미션:** `input:focus` (집중) 상태일 때,
    1.  `border-color: #3498db;` (파란색)로 테두리 색을 바꾸세요.
    2.  `outline: none;`을 줘서 브라우저 기본 테두리(보통 파란색 아웃라인)를 제거하세요.
  * **평가 기준 (Rubric):**
      * [A] `:focus` 가상 클래스 선택자를 사용함.
      * [B] `border-color`와 `outline: none;` 속성을 모두 적용함.
      * [C] `input` 클릭 시, 테두리만 파란색으로 깔끔하게 변경됨.

**문제 10: 긴급 전송 버튼 (Button Styling)**

  * '전송' `<button>`이 못생겼습니다.
  * **미션:**
    1.  `width: 100%`, `padding: 15px` 적용.
    2.  배경색 `background: #e74c3c;` (빨간색), 글자색 `color: white;` 적용.
    3.  `border: none;` (테두리 없음), `border-radius: 5px;` (모서리 둥글게) 적용.
    4.  마우스를 올리면(`:hover`) 배경색이 `background: #c0392b;` (더 진한 빨간색)로 변하게 하세요.
  * **평가 기준 (Rubric):**
      * [A] `button`의 기본 스타일(테두리, 배경)을 초기화하고 새 스타일을 적용함.
      * [B] `border-radius`로 모서리를 둥글게 처리함.
      * [C] `:hover` 상태의 시각적 변화가 명확하게 구현됨.

### 3\. 🏗️ 우주 기지 설계도 (Semantic & Complex Layout)

**문제 11: `div`로 지은 집 (Semantic Conversion)**

  * 사이트가 `div`로만 지어져 있습니다.
    ```html
    <div id="logo-area">...</div>
    <div class="menu">...</div>
    <div id="main-content">...</div>
    <div id="copyright">...</div>
    ```
  * **미션:** 위 HTML 코드를 `id`나 `class` 이름에 맞게 시맨틱 태그(`<header>`, `<nav>`, `<main>`, `<footer>`)로 변경하세요.
  * **평가 기준 (Rubric):**
      * [A] 4개의 `div`를 각각 `header`, `nav`, `main`, `footer`로 정확히 대체함.
      * [B] 불필요한 `id`나 `class` (예: `id="main-content"`)를 제거함.
      * [C] 페이지의 의미론적 구조가 명확해짐.

**문제 12: 본문과 사이드바 (Article & Aside)**

  * '메인 콘텐츠'(`<main>`) 영역 안에 '주요 뉴스'와 '관련 링크'가 섞여 있습니다.
  * **미션:** `<main>` 태그 안에,
    1.  '주요 뉴스' 영역을 `<article>` 태그로 감싸세요.
    2.  '관련 링크' 영역을 `<aside>` 태그로 감싸세요.
  * **평가 기준 (Rubric):**
      * [A] `<main>` 태그 내부에 `<article>`과 `<aside>`를 배치함.
      * [B] 각 태그의 의미(본문, 부가 정보)에 맞게 콘텐츠를 분리함.

**문제 13: 이름표(태그)로 스타일링하기 (Tag Selectors)**

  * 시맨틱 태그로 구조를 바꿨으니, CSS도 `id`나 `class` 대신 '이름표'(태그)로 스타일링합니다.
  * **미션:**
    1.  `<header>` 태그는 `background-color: #1a2533;` (우주 검정)로 설정하세요.
    2.  `<footer>` 태그는 `background-color: #333;` (진회색)과 `text-align: center;` (가운데 정렬)로 설정하세요.
  * **평가 기준 (Rubric):**
      * [A] `.`(class)이나 `#`(id) 없이 `header`, `footer` 태그 선택자를 직접 사용함.
      * [B] 각 영역에 요청된 스타일이 정확히 적용됨.

**문제 14: 레이아웃에 테이블 넣기 (Semantic + Table)**

  * '우주 기지 설계도'(`Semantic`)가 완성되었습니다. `<main>` 영역 안에 '행성 데이터'(`Table`)를 넣어야 합니다.
  * **미션:**
    1.  `<main>` 태그를 만드세요.
    2.  그 안에 [문제 3]에서 만들었던 '얼룩말 무늬' `<table>` 코드를 그대로 삽입하세요.
  * **평가 기준 (Rubric):**
      * [A] HTML 구조가 `<main>` 안에 `<table>`이 중첩된 형태임.
      * [B] CSS가 충돌하지 않고, 테이블의 '얼룩말 무늬' 스타일이 `<main>` 안에서도 정상적으로 적용됨.

**문제 15: 최종 대시보드 (Final Boss: All Concepts)**
